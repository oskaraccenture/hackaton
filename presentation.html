The Patient — spring-music

What we inherited

  • Multi-annotation entity — Album.java carries @Entity, @Document, @RedisHash simultaneously
  • Cloud Foundry lock-in — manifest.yml + end-of-life spring-cloud-connectors
  • Bundled frontend — AngularJS compiled into the JAR, no independent deployment
  • Zero API contract — no OpenAPI spec; consumers depend on observed behavior
  • No characterization tests — behavior is completely unpinned

The board's ask

  • Remove Cloud Foundry dependency
  • Enable independent service deployment
  • Explicit API contract for consumers
  • Testable — before and after every change

Stack

  • Legacy: Spring Boot 2.x, Gradle, H2/MySQL/MongoDB/Redis
  • New service: Node.js + Express, SQLite in-memory

The Strategy — Strangler Fig

Extract one service at a time. The monolith stays live. Each service is independently provable before any traffic shift.

Key Decisions (ADRs)

ADR-001 — Strangler Fig

Monolith stays live. Extract one service at a time. Characterization tests pin behavior before any extraction.

ADR-002 — spring-music as legacy (option B)

Real code, real anti-patterns (Album.java carries JPA+MongoDB+Redis annotations), genuine CF coupling. More interesting than generated ugliness.

ADR-003 — PreToolUse hook vs. CLAUDE.md prompt

Hook = deterministic hard block for Spring annotations leaking into service routes.
Prompt = probabilistic preference for using the ACL adapter.
Rule: if failure mode is silent contract violation → hook. If suboptimal but correctable → prompt.

How We Used Claude Code

Where it saved the most time

  • Legacy archaeology — call-graph analysis of AbstractAlbumRepository in minutes
  • Characterization tests — pinned HTTP behavior including known quirks faster than writing manually
  • ADR drafting — bullet list → full ADR with "what we chose NOT to do" sections
  • CLAUDE.md-driven boundaries — taught ACL rule once, respected across all generations

Techniques we reached for

  • 3-level CLAUDE.md (project / legacy / services)
  • PreToolUse hook for deterministic boundary enforcement
  • Custom slash commands: /extract-service, /characterize, /write-adr
  • Plan Mode before every multi-file extraction

Biggest surprise: Claude identified the multi-DB profile pattern as the primary coupling risk — not the entity model, which we assumed going in.
